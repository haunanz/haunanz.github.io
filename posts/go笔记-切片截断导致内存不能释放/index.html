Go笔记 切片截断导致内存不能释放

引言

go与语言在使用切片截断时，底层数组没有改变，依然持有指向堆内存的指针，导致内存无法释放。

// 示例
type Node [100]byte

data := []*Node{}
for i:=10; i<10; i++ {
  tmp := &Node{}
  data = append(data,tmp)
}

data = data[:len(data)-1] // 此时由于底层数组没有改变 所以内存不会释放

切片的数据结构

type arr struct {
   Data unsafe.Pointer
   Len  int
   Cap  int
}

切片截断只是改变了Len字段，并没有改变底层数组。

解决方法

解决方法，是在截断前手动置空。


data[len(data)-1]=nil
data = data[:len(data)-1]
