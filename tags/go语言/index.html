  • 经典同步问题go语言实现

    2020-11-18 GO语言

    并发编程中各个并发实例有许多的制约关系，为了协调各个实例的制约关系，引入了同步的概念。
    下面了解几个经典的同步问题：

    1. 生产者消费者问题
    2. 读写者问题
    3. 哲学家就餐问题
    4. 吸烟者问题 许多的同步问题都可以简化为这三类问题。

    基本概念

    解决这些问题只需要了解互斥这个概念就可以做到，下面来了解一些基本概念。

    临界资源： 系统中的某些资源一次只能有一个并发实例使用，多个实例使用时是未定义的。
    临界区：并发实例中访问临界资源的那段代码。
    同步：多个并发实例为了协调工作次序而形成的相互制约关系。
    互斥：当一个并发实例进入临界时，其它并发实例会被阻塞。
    阻塞： 并发实例因为不能获得某些资源而等待的行为。
    互斥锁： 一般是由硬件系统和操作系统共同提供的接口，来帮助用户实现互斥的工具。
    死锁： 多个并发实例占有一个临界资源，但是又需要被别的实例占有的某个资源。互相期望对方的资源。导致程序不能执行下去。
    饥饿： 只一个实例长期得不到临界资源的现象。 原子操作: 又被叫做原语,只操作过程中连续的,不可被中断的一组程序。

    特别的：
    互斥锁一般由原子操作来实现。
    硬件系统直接提供了swap指令来直接支持，因为该指令要占用总线吗，所以在多核CPU中也有效。

    go语言在 sync 包 提供了 sync.Mutex 互斥锁。 c语言提供了 <pthread.h> 头文件中提供了 pthread_mutex_t 互斥锁。

    import "sync"

mutx := sync.Mutex{}  // 创建互斥锁
mutx.Lock()           // 锁住临界区
// do something
mutx.Unlock()         // 释放临界区

    可以了解一下在C语言的中的实现,往往会用到😂。

    // c语言相关(可以忽略)
 // 环境是在linux中 
 // 如果使用gcc请注意安装时的线程实现方式选项选POSIX

// do something

  • 为什么要使用动态内存

    2020-10-26 GO语言

    如果在编码时就可以确定作用域和所需内存大小，那么动态内存就没有必要使用。
    但是在实际的生产中，对于复杂的系统来说，内存的大小总是不可估计的。

    下面总结了三种需要动态分配内存的情型：

    1. 不清楚所需的类型的数量，如：

    // 传入给定的字符 要求所有字符的哈希
func mHash(strs []string) []int{
  // do something
  // return asn
}

    不清楚传入的参数有多大，也就不清楚传出的参数有多大，需要动态分配。
    如果申请一个很大的数组也可以，但是太浪费内存了。
    为了节约内存，使用动态内存模型是必然的，虽然付出了一点时间开销，但是是值得的，在很多时候也是必须的。

    2. 不清楚所需的数据的具体类型。

    如下面的两个struct都实现了dater接口。
    producer根据参数返回指定的接口类型，那么函数该在栈上分配多大的空间呢?
    所以对于不清楚的数据类型都是在堆上分配。

    type dater interface {
  Date()
}

type time1 struct{
  a int
}

func (time1)Date(){}

type time2 struct{}

func (time2)Date(){}


func producer(name string ) dater {
  if name == "" {
    return time1{}
  }
  return time2{}
}

    3. 需要共享数据。

    在栈上分配内存时，内存在离开作用域后析构。
    那么，就只能在生命周期更长的作用域申请，如果软件都这样实现，增加了软件的实现复杂度。

  • Go语言Linux网络编程 使用原始套接字

    2020-10-26 网络编程 GO语言

    同时监听TCP和UDP

    面试常见题，是否可以在一个linux的网络端口同时处理TCP和UDP。
    常间的同时处理TCP和UDP的程序有超级服务inetd 和安卓的调试服务 adbd。

    一个套接字只能绑定一个端口，如果一个端口要同时处理两种服务，就要申请两个套接字来监听一个端口。一个流式套接字，一个数据报套接字。

    示例

    package main


import (
	"fmt"
	"io"
	"log"
	"net"
	"os"
)

// 错误处理
func doErr(err error) {
	if err != nil {
		log.Println(err)
		os.Exit(1)
	}
}

// 处理tcp
func handle(conn net.Conn) {
	defer conn.Close()
	var buf [125]byte
	for {
		n, err := conn.Read(buf[:])
		if err == io.EOF {
			break
		} else {
			doErr(err)
		}

		_, err = conn.Write(buf[:n])
		doErr(err)
	}
}

func main() {
  // tcp listen
	tcpListen, err := net.Listen("tcp", ":8086")
	doErr(err)
	defer tcpListen.Close()

  // udp listen
	udpAddr, _ := net.ResolveUDPAddr("udp", ":8086")
	var udpListen *net.UDPConn
	udpListen, err = net.ListenUDP("udp", udpAddr)
	doErr(err)
	defer udpListen.Close()

  // 处理 udp 
	go func() {
		for {
			var buf [125]byte
			n, addr, err := udpListen.ReadFromUDP(buf[:])
			doErr(err)
			_, err = udpListen.WriteToUDP(buf[:n], addr)
			doErr(err)
		}
	}()

  // 处理 tcp
	for {
		conn, _ := tcpListen.Accept()
		go handle(conn)
	}
}

    测试： $ nc 0.0.0.0 8086 # tcp测试
    $ echo "some message" | nc -4u 0.0.0.0 8086 # upd测试

  • Go语言切片截断导致内存泄露

    2020-09-26 GO语言

    引言

    go与语言在使用切片截断时，底层数组没有改变，依然持有指向堆内存的指针，导致内存无法释放。

    // 示例
type Node [100]byte

data := []*Node{}
for i:=10; i<10; i++ {
  tmp := &Node{}
  data = append(data,tmp)
}

data = data[:len(data)-1] // 此时由于底层数组没有改变 所以内存不会释放

    切片的数据结构

    type arr struct {
   Data unsafe.Pointer
   Len  int
   Cap  int
}

    切片截断只是改变了Len字段，并没有改变底层数组。

    解决方法

    解决方法，是在截断前手动置空。

    data[len(data)-1]=nil
data = data[:len(data)-1]

  • Go原始套接字编程

    2020-09-01 网络编程 GO语言

    请注意，本文档的运行环境是Linux。

    在Windows环境下注意系统调用的接口有差别。

    两个计算机如何在网络中定位彼此的位置呢？计算机中的进程如何在网络中定位和自己对话的进程呢？ 通过套接字就可以唯一定位两个通信的进程。
    套接字一般来说就是： IP地址 + 端口号。
    IP地址唯一确定计算机的地址，端口号唯一确定计算机中的一个进程。 IPv4约有40亿个地址。每个计算机系统的端口号从0-65535。（当然可以通过配置网卡来获得多个地址，但每个地址接口的上限都是65535）

    套接字由操作系统提供和维护，当程序向系统申请时，系统会返回一个文件描述符。通过这个描述符就可以进行通信相关的一系列操作。

    一般来说，使用套接字进行通信时，由一方扮演服务方，另一方扮演客户方。
    服务方往往通过监听端口被动的接收连接，通过单个端口可以服务成千上万个连接。
    客户方往往主动发起连接请求，每个连接都要消耗一个系统的端口号。

    Socket 向系统申请一个套接字

    申请套接字的这一步操作是为了让系统根据你设置的协议类型对套接字进行相应的初始化。
    进行这个操作后，系统将会为你分配一个套接字，返回给你一个文件描述符。你可以使用这个文件描述符操作这个套接字。

    syscall.Socket

    地址族

    地址族取值描述
    AF_UNIX 或者 AF_LOCAL本地通信
    AF_INETIPv4
    AF_INET6IPv6

    套接字类型

    套接字类型描述
    SOCK_STREAM面向字节流的，用于TCP协议
    SOCK_DGRAM面向数据报，用于UDP协议
    SOCK_RAW原始套接字，用于IP，ICMP等更加底层的协议

    协议类型

    协议类型描述
    IPPROTO_TCPTCP协议
    IPPROTO_UDPUDP协议

    Listen 监听端口

    服务器如何知晓外部想和自己建立连接呢？
    通过监听对应的端口。

    Accept 接收并建立连接

    通过监听，我们知道-有外部连接请求。如何描述对方呢？换句话说，如果我们接收了很多连接请求，如何区分这些连接呢？
    我们想操作这些连接如何指定对象呢？
    答案是套接字！和监听套接字一样的套接字。
