    同时监听TCP和UDP

    面试常见题，是否可以在一个linux的网络端口同时处理TCP和UDP。
    常间的同时处理TCP和UDP的程序有超级服务inetd 和安卓的调试服务 adbd。

    一个套接字只能绑定一个端口，如果一个端口要同时处理两种服务，就要申请两个套接字来监听一个端口。一个流式套接字，一个数据报套接字。

    示例

    package main


import (
	"fmt"
	"io"
	"log"
	"net"
	"os"
)

// 错误处理
func doErr(err error) {
	if err != nil {
		log.Println(err)
		os.Exit(1)
	}
}

// 处理tcp
func handle(conn net.Conn) {
	defer conn.Close()
	var buf [125]byte
	for {
		n, err := conn.Read(buf[:])
		if err == io.EOF {
			break
		} else {
			doErr(err)
		}

		_, err = conn.Write(buf[:n])
		doErr(err)
	}
}

func main() {
  // tcp listen
	tcpListen, err := net.Listen("tcp", ":8086")
	doErr(err)
	defer tcpListen.Close()

  // udp listen
	udpAddr, _ := net.ResolveUDPAddr("udp", ":8086")
	var udpListen *net.UDPConn
	udpListen, err = net.ListenUDP("udp", udpAddr)
	doErr(err)
	defer udpListen.Close()

  // 处理 udp 
	go func() {
		for {
			var buf [125]byte
			n, addr, err := udpListen.ReadFromUDP(buf[:])
			doErr(err)
			_, err = udpListen.WriteToUDP(buf[:n], addr)
			doErr(err)
		}
	}()

  // 处理 tcp
	for {
		conn, _ := tcpListen.Accept()
		go handle(conn)
	}
}

    测试： $ nc 0.0.0.0 8086 # tcp测试
    $ echo "some message" | nc -4u 0.0.0.0 8086 # upd测试

  • Go原始套接字编程

    2020-09-01 网络编程 GO语言

    请注意，本文档的运行环境是Linux。

    在Windows环境下注意系统调用的接口有差别。

    两个计算机如何在网络中定位彼此的位置呢？计算机中的进程如何在网络中定位和自己对话的进程呢？ 通过套接字就可以唯一定位两个通信的进程。
    套接字一般来说就是： IP地址 + 端口号。
    IP地址唯一确定计算机的地址，端口号唯一确定计算机中的一个进程。 IPv4约有40亿个地址。每个计算机系统的端口号从0-65535。（当然可以通过配置网卡来获得多个地址，但每个地址接口的上限都是65535）

    套接字由操作系统提供和维护，当程序向系统申请时，系统会返回一个文件描述符。通过这个描述符就可以进行通信相关的一系列操作。

    一般来说，使用套接字进行通信时，由一方扮演服务方，另一方扮演客户方。
    服务方往往通过监听端口被动的接收连接，通过单个端口可以服务成千上万个连接。
    客户方往往主动发起连接请求，每个连接都要消耗一个系统的端口号。

    Socket 向系统申请一个套接字

    申请套接字的这一步操作是为了让系统根据你设置的协议类型对套接字进行相应的初始化。
    进行这个操作后，系统将会为你分配一个套接字，返回给你一个文件描述符。你可以使用这个文件描述符操作这个套接字。

    syscall.Socket

    地址族

    地址族取值描述
    AF_UNIX 或者 AF_LOCAL本地通信
    AF_INETIPv4
    AF_INET6IPv6

    套接字类型

    套接字类型描述
    SOCK_STREAM面向字节流的，用于TCP协议
    SOCK_DGRAM面向数据报，用于UDP协议
    SOCK_RAW原始套接字，用于IP，ICMP等更加底层的协议

    协议类型

    协议类型描述
    IPPROTO_TCPTCP协议
    IPPROTO_UDPUDP协议

    Listen 监听端口

    服务器如何知晓外部想和自己建立连接呢？
    通过监听对应的端口。

    Accept 接收并建立连接

    通过监听，我们知道-有外部连接请求。如何描述对方呢？换句话说，如果我们接收了很多连接请求，如何区分这些连接呢？
    我们想操作这些连接如何指定对象呢？
    答案是套接字！和监听套接字一样的套接字。
    还记得我们在文章开始说过吗：IP地址：端口 唯一确定在网络中的主机上运行的一个进程。
    那么如何获得这个套接字呢？ Accept 其实自己也可以创建，但是操作系统帮我们处理了很多底层的东西，降低了实现网络通信的的复杂度。
